What CodeSync doesn't have

You've never estimated correctly. Nobody has. We track actual cycle time instead. Radical, we know.
Removed

Removed
Priority dropdowns
Position in the list IS the priority. Drag it up if it matters. Revolutionary technology from the year 1985.

Velocity charts
You know what's a good velocity metric? Shipped features. Check your git log.

What it actually does

01
AI executes your tasks
Click "Run" on a task. Claude Code reads your codebase, plans the work, implements it, runs tests, and presents you with a review. You approve or reject. That's the workflow.
plan → implement → verify → review
02
One task. Right now.
The focus view shows you the single most impactful task based on what's blocked, what's due, and what you've prioritized. Not a board with 47 columns. One thing.
03
It asks when it's stuck
If Claude doesn't know something, the job pauses and asks you. Answer inline from the UI or from Claude Code via MCP. It picks up where it left off.
04
Git integration that isn't a footgun
Claude never touches git. When you approve a task, you choose: commit, commit & push, or create a branch + PR. Predefined commands. No improvisation.
05
Checkpoints & revert
Before every task, CodeSync snapshots your working directory. If the AI breaks something, revert to exactly where you were. Failed tasks auto-revert.
06
Runs on your machine
Your code never leaves your laptop. CodeSync is a local server + web UI. Supabase runs in Docker for data. Team sync optional. Zero cloud dependency for solo devs.
07
Human tasks too
Not everything is for AI. Toggle a task to "human mode" and it's just a task. Start, done, move on. Design work, meetings, whatever — it all lives in one place.
08
MCP server included
Ask Claude Code "what should I work on?" and it reads your project board. Create tasks, reply to paused jobs, approve reviews — all from the CLI.

Three commands to sanity

1
Clone & start
Needs Node, Docker, and a Claude subscription. That's the whole dependency list.
git clone git@github.com:ilyador/codesync.git && cd codesync && pnpm install
2
Start Supabase + dev server
Supabase runs locally in Docker. Your data stays on your machine.
npx supabase start && npx supabase db reset && pnpm dev
3
Open, create a project, ship
Point it at your codebase. Create a task. Click Run. Go get coffee. Come back to a PR.
open http://localhost:3000
