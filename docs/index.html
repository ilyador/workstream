Your custom worker
Whatever you can describe
AI designer from Figma screenshots. Copywriter for release notes. Security auditor against OWASP. Hook up RAG with local embeddings and they can search your docs before writing code.

Every worker is a composable flow. Drag steps, switch models, control context per step. Write shared instructions in agents.md that shape the worker's personality. The flow editor is visual. No YAML. No config files. No PhD in prompt engineering.

What it does instead

01
Assign like a teammate
One dropdown: pick an AI worker or a human. "Who does this?" That's the whole decision. No mode toggles, no configuration wizards.
02
Streams, not boards
Tasks run top-to-bottom. One finishes, next starts. No dragging cards across five columns so a PM can feel informed.
03
They ask when stuck
Job pauses. You answer inline. It continues. Unlike your last team lead, it doesn't pretend to understand and break everything.
04
Git is automatic
Each stream gets a worktree. Each task auto-commits. Checkpoint before execution, auto-revert on failure. Click "Create PR" when done. Zero git decisions.
05
Watch them work live
Streaming logs as Claude reads files, edits code, runs tests. Push notifications when done. Not 2004-style dashboard refreshing.
06
Humans welcome too
Design reviews, QA, meetings -- assign to a person. AI and humans share the same board. Comments, @mentions, the whole thing.
07
Phone, terminal, browser
Telegram bot for your phone. MCP server for your terminal. Web UI for your browser. Manage your AI team from wherever you are.

Two ways to run it

On your machine
Run locally, sync through an online Supabase. Good for solo developers who want AI as a coding partner while they work.
On a VPS
Run on a server with local Supabase. AI grinds tasks 24/7 while you sleep. Good for teams and background automation.

Each workstream gets its own git worktree — your main branch stays clean, multiple workstreams run concurrently. Each task auto-commits. When a workstream is done, click Create PR.

Three steps no onboarding call

1
Clone it
Needs Node, Docker, and a Claude subscription. That's the whole dependency list.
git clone git@github.com:ilyador/workstream.git && cd workstream && pnpm install
2
Start it
Starts Supabase and the dev server. One command, two services.
npx supabase start && npx supabase db reset && pnpm dev
3
Ship something
Point it at your repo. Create a task. Click Run. Go make coffee. Come back to a PR.
open http://localhost:3000
